Voici un récapitulatif de toutes les commandes liées à Nx Witness (Network Optix) avec leur utilité :

Installation

Télécharge le paquet d'installation du serveur.

Installe le paquet en gérant automatiquement les dépendances.

bash
sudo apt --fix-broken install -y
Corrige les dépendances manquantes si l'installation échoue.

Méthodes alternatives d'installation (moins recommandées) :

bash
sudo apt install gdebi-core -y
sudo gdebi nxwitness-server.deb
bash
sudo dpkg -i nxwitness-server.deb
sudo apt --fix-broken install -y
Gestion du service

sudo systemctl status networkoptix-mediaserver
Affiche l'état du service (actif, arrêté, en erreur).

bash
sudo systemctl start networkoptix-mediaserver
Démarre le service.

bash
sudo systemctl stop networkoptix-mediaserver
Arrête le service.

bash
sudo systemctl restart networkoptix-mediaserver
Redémarre le service (utile après une modif de config).

bash
sudo systemctl enable networkoptix-mediaserver
Active le démarrage automatique au boot du serveur.

Diagnostic / Logs
bash
sudo journalctl -u networkoptix-mediaserver -n 100 --no-pager
Affiche les 100 dernières lignes de log du service (essentiel pour diagnostiquer un crash).

sudo journalctl -u networkoptix-mediaserver -f
Suit les logs en temps réel (-f = follow).

Affiche le log interne propre à Nx Witness (différent du log système).

Localise le fichier de log si le chemin par défaut a changé.

bash
sudo /opt/networkoptix/mediaserver/bin/mediaserver-bin
Lance le serveur manuellement en avant-plan (hors service systemd) pour voir l'erreur exacte en direct — très utile en cas de crash silencieux.
